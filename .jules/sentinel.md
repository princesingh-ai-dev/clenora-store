## 2024-05-15 - [DOM-based XSS Mitigation via localStorage and Event Handlers]
**Vulnerability:** The application was directly interpolating data fetched from `localStorage` (`clenoraCart`) into `innerHTML` to render the shopping cart and checkout summary. Additionally, it passed item variables into inline `onclick` event handlers. This created a DOM-based Cross-Site Scripting (XSS) vulnerability, as an attacker controlling `localStorage` could execute arbitrary JavaScript.
**Learning:** `localStorage` must be treated as untrusted user input just like query parameters or API responses. Furthermore, inline HTML event handlers (e.g., `onclick="..."`) unescape HTML entities before execution, meaning simply escaping HTML is insufficient to prevent XSS if the data is placed inside an inline event handler attribute. Finally, when creating custom HTML sanitization functions, input must be explicitly cast to a string (e.g., `String(str)`) to prevent array bypass vulnerabilities where a malicious array acts as an object and bypasses string-specific regex replacements.
**Prevention:** Always sanitize data retrieved from `localStorage` before injecting it into the DOM. Use `escapeHTML` (with explicit string casting) for text/attribute contexts, and avoid inline event handlers entirely when dealing with dynamic data. Instead, use data attributes (`data-id`) and event delegation. Also, when retrieving data from the DOM via attributes (which are always strings), explicitly cast internal variables to strings (e.g., `String(item.id)`) before comparison to avoid type mismatch bugs.

## 2024-05-10 - [Stored XSS via LocalStorage]
**Vulnerability:** Found a Cross-Site Scripting (XSS) vulnerability related to `localStorage` caching of cart items. `app.js` renders `cart` items using `innerHTML` without sanitizing input properties such as `item.name`, `item.image`, and `item.id`. This data persists in `localStorage` under the key `clenoraCart`. If an attacker can inject malicious strings into `localStorage` directly, or if there's another vulnerability that writes unsanitized data into `localStorage`, the payload will be repeatedly executed whenever `updateCartDisplay` or `renderOrderSummary` are called.
**Learning:** `localStorage` is not an inherently safe origin for data. Any data read from `localStorage` that is subsequently rendered directly into the DOM (e.g., using `innerHTML`) without sanitization poses a severe XSS risk. Always sanitize cached/persisted state, especially if the structure implies user-influenced inputs, even if initially loaded locally.
**Prevention:** Always use an `escapeHTML` helper or safer DOM APIs (like `textContent` instead of `innerHTML` or `document.createElement`) when reflecting data from persistent storage back into the DOM. When using template literals with `innerHTML`, explicitly escape every interpolation of dynamic data. Number coercions can also be used for purely numeric fields like `price` and `quantity` to further prevent injections.
